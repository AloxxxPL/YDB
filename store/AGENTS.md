# Store — Zustand State Management

## Overview

The `store/` directory contains two Zustand stores: `useAppStore` (persisted) and `useAuthStore` (ephemeral).

The **primary store** is `useAppStore` — it persists all application state to AsyncStorage and is used across the entire app. `useAuthStore` is a legacy remnant and largely unused.

---

## `app.ts` — `useAppStore` (Persisted)

### Configuration

- **Storage:** AsyncStorage (React Native)
- **Persistence key:** `'app-store'`
- **Lifecycle:** Data survives app restart and kill

### State Fields

| Field | Type | Initialized | Purpose |
|---|---|---|---|
| `isReady` | `boolean` | `false` | Flags app boot completion; used to orchestrate initialization logic |
| `token` | `string \| null` | `null` | JWT session token (currently unused; planned for auth integration) |
| `userId` | `string \| null` | `null` | Current user ID (set by onboarding → dishes form) |
| `profile` | `Profile \| null` | `null` | Full user profile from Supabase, synced after onboarding |
| `tempProfile` | `TempProfile` | `{}` | Ephemeral data collected during onboarding; cleared after `formsCompleted` |
| `formsCompleted` | `boolean` | `false` | Gate flag: if `false`, `/forms/name` is shown; if `true`, app proceeds to home |
| `dietPlan` | `DietPlan \| null` | `null` | Current meal plan (generated by Gemini); persisted and survives app restart |
| `dietLoading` | `boolean` | `false` | Flags if diet generation is in progress |
| `dietError` | `string \| null` | `null` | Error message from failed diet generation; cleared on successful generation |

### Actions

| Action | Signature | Used By |
|---|---|---|
| `setReady(ready)` | `(ready: boolean) => void` | `_layout.tsx` (init phase) |
| `setToken(token)` | `(token: string \| null) => void` | (Auth integration planned) |
| `setUserId(userId)` | `(userId: string \| null) => void` | `dishes.tsx` (on profile save) |
| `setProfile(profile)` | `(profile: Profile \| null) => void` | `dishes.tsx` (on profile save) |
| `setTempProfile(tempProfile)` | `(tempProfile: TempProfile) => void` | (Rarely used directly) |
| `updateTempProfile(partial)` | `(partial: Partial<TempProfile>) => void` | All form steps: `name.tsx`, `gender.tsx`, `age.tsx`, `height.tsx`, `weight.tsx`, `goal.tsx`, `dishes.tsx` |
| `setFormsCompleted(completed)` | `(completed: boolean) => void` | `dishes.tsx` (on confirmation) → unlocks the `index.tsx` gate |
| `setDietPlan(plan)` | `(plan: DietPlan \| null) => void` | `dishes.tsx` (after generation), `diet.tsx` (after refinement) |
| `setDietLoading(loading)` | `(loading: boolean) => void` | `dishes.tsx`, `diet.tsx` (during async operations) |
| `setDietError(error)` | `(error: string \| null) => void` | `dishes.tsx`, `diet.tsx` (error handling) |

### Key Patterns

**Shallow merge in `updateTempProfile()`:**
```ts
updateTempProfile: (partial) =>
  set((state) => ({
    tempProfile: { ...state.tempProfile, ...partial },
  }))
```
Each form writes only its own field — persisting the others. Enables form re-entry with partial data intact.

**Persistence contract:**
- **Persisted fields:** All except `token` (currently unused)
- **Survives:** App kill, OS suspend, navigation
- **Cleared:** Never (explicit cleanup not implemented; can be done via `setFormsCompleted(false)` if needed)

## `auth.ts` — `useAuthStore` (Ephemeral)

### Configuration

- **Storage:** In-memory only (no persistence)
- **Lifecycle:** Cleared on app kill

### State Fields

| Field | Type | Initialized | Purpose |
|---|---|---|---|
| `token` | `string \| null` | `null` | JWT token (unused; for future auth) |
| `userId` | `string \| null` | `null` | User ID (unused; use `useAppStore.userId` instead) |
| `isAuthenticated` | `boolean` | `false` | Auth flag (unused; for future auth) |

### Actions

| Action | Signature | Used By |
|---|---|---|
| `setToken(token)` | `(token: string) => void` | (Unused) |
| `clearAuth()` | `() => void` | (Unused) |

### Status

⚠️ **Legacy / Unused:** This store predates the current architecture. All user data is managed in `useAppStore`. Do not add new dependencies on `useAuthStore`; use `useAppStore` instead.

## Data Flow — Onboarding to Home

```
Step 1–6 (forms)
  └─ Each form: updateTempProfile({ fieldName: value })
     └─ Zustand updates tempProfile in-memory + AsyncStorage

Step 7 (dishes.tsx) — confirm()
  ├─ supabase.insert(profileData)
  ├─ setUserId(tempUserId)
  ├─ setProfile(profileData)
  ├─ setFormsCompleted(true)  ← unlocks gate
  ├─ generateDietPlan() [background]
  └─ router.replace('/')

Next app restart:
  ├─ AsyncStorage rehydrates useAppStore
  ├─ formsCompleted === true → index.tsx gate passes
  └─ dietPlan rendered (or loading shown)
```

---

## Hydration & Initialization

**In `app/_layout.tsx`:**
1. App mounts → `_layout.tsx` loads
2. Zustand automatically rehydrates from AsyncStorage (due to `persist` middleware)
3. `setReady(true)` called manually after initial check

**Re-entry behavior:**
- If `formsCompleted === false` → redirect to `/forms/name`
- If `formsCompleted === true` → render home screen
- `tempProfile` is cleared conceptually (not needed after `formsCompleted`), but remains in AsyncStorage for potential recovery

---

## Testing Considerations

**To reset state for testing:**
1. Clear AsyncStorage manually via Expo CLI or app settings
2. Or: call `setFormsCompleted(false)` + `setTempProfile({})` to re-trigger onboarding

**To inspect persisted state:**
- AsyncStorage key: `'app-store'`
- Value is JSON-stringified, can be read via console or React DevTools
